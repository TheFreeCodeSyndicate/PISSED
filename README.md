
PISSED

The purpose of this repository is to maintain a growing catalogue of software that members of The Freecode Syndicate find frustrating, limiting, expensive, inaccessible, unnecessarily complicated, poorly designed, abandoned, over-commercialised, or simply offensive to the human spirit.

----- 


What Goes Here?

Anything.

Well, almost anything.

You can submit:

- Software you dislike.
- Services you wish were open source.
- Tools that became unusable after an update.
- Products trapped behind subscriptions.
- Applications that are unnecessarily complicated.
- Software that respects neither your time nor your hardware.
- Platforms that should not require an internet connection but somehow do.
- Programs that consume 2 GB of RAM to display what could have been a text file.
- Tools that you secretly wish you had written yourself.

When possible, explain:

> What the software does.
> Why it annoys you.
> What specific problems you encounter.
> What features are missing.
> What an alternative might look like.

A useful complaint is worth more than an angry one.

Contributing

Contributions are welcome. In fact, contributions are the entire point. If a piece of software has irritated you enough that you spent twenty minutes explaining the problem to a friend, there is a reasonable chance it belongs here.

[GUIDELINES FOR ADDING TO PISSED LIST: ]

Before opening a pull request:

> Check whether the software already exists in the repository.
> If it does, improve the existing entry instead of creating a duplicate.
> Explain why the software belongs here.
> Remember that developers are people too, even when their software has ruined your afternoon.

[GUIDELINES FOR PROPOSING BETTER SOFTWARES ALTERNATIVES, PROJECTS IDEAS TO MAKE IT OURSELVES ETC]

If you identify software that deserves a better alternative, consider creating a proposal that includes:

> Core functionality
> Features to keep
> Features to remove
> Licensing ideas
> Technical considerations
> Potential implementation strategy
